# Security Policy

LAI is currently experimental and should not be considered production-ready.

## Reporting a vulnerability

- Please do not publish sensitive security details in a public issue.
- Prefer a private GitHub security advisory if the repository enables it.
- If private reporting is not yet configured, contact the maintainers directly through GitHub before disclosing details publicly.

## Scope

Security reports are especially helpful for:

- native iOS code execution paths
- model loading and asset handling
- unsafe file access or bundle handling
- dependency vulnerabilities with realistic impact
